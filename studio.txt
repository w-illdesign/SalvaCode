<section>
  <h2 class="sous-titre">Pour ceux qui ont un ordinateur</h2>

  <p><b>Avant de commencer — quelques mots simples :</b></p>
  <p>
    Imagine ton ordinateur comme une maison :
    <ul>
      <li>Un <b>dossier</b> est comme une <i>chemise</i> où tu ranges des feuilles (fichiers).</li>
      <li>Un <b>fichier</b> est comme une <i>feuille de papier</i> avec du texte (par ex. <code>main.py</code>).</li>
      <li>Le <b>terminal</b> (invite de commandes) est comme parler au concierge : tu lui dis ce que tu veux qu’il fasse et il l’exécute.</li>
      <li><b>VS Code</b> est ton <i>atelier</i> ou ton <i>bureau</i> où tu écris ton code, tu ranges tes outils, et tu exécutes des tâches plus facilement.</li>
    </ul>
  </p>

  <h3>1. Installer Python — qu’est-ce que c’est ?</h3>
  <p>
    Python est le langage que l’ordinateur comprend pour exécuter les instructions que tu écris.
    Pense à Python comme à une machine à café : tu lui donnes une recette (ton code) et elle te rend la boisson (le résultat).
  </p>

  <ol>
    <li>
      <b>Télécharger</b> Python :
      <br>
      Va sur <a href="https://www.python.org/downloads/" target="_blank" rel="noopener">python.org/downloads</a> et clique sur <b>Download Python 3.x</b>.
    </li>

    <li>
      <b>Exécuter l’installeur</b> :
      <ul>
        <li>Windows : double-clic sur le fichier téléchargé (.exe).</li>
        <li>macOS : ouvre le paquet .pkg et suis les instructions.</li>
        <li>Linux : tu peux utiliser le gestionnaire de paquets (exemples plus bas).</li>
      </ul>
    </li>

    <li>
      <b>Important — Windows :</b> coche <b>Add Python to PATH</b> avant d’installer.
      <p><i>Analogie :</i> PATH = une liste de chemins que le concierge (terminal) vérifie. Si Python n’est pas dans la liste, le concierge ne saura pas où le trouver.</p>
    </li>

    <li>
      <b>Vérifier</b> :
      <div class="cmd-row">
        <pre class="cmd language-bash"><code>python --version
# ou, si la première ne marche pas :
python3 --version
# sur certaines installations Windows :
py -3 --version</code></pre>
      </div>
      <p>Tu dois voir quelque chose comme : <code>Python 3.x.x</code>.</p>
    </li>

    <li>
      <b>Commandes utiles (Linux)</b> :
      <div class="cmd-row">
        <pre class="cmd language-bash"><code>sudo apt update
sudo apt install -y python3 python3-pip</code></pre>
      </div>
    </li>
  </ol>

  <h3>2. Installer Git (et Git Bash)</h3>
  <p>
    Git sert à garder l’historique de ton travail — comme un journal où tu notes chaque modification. Git Bash est un terminal agréable sur Windows.
  </p>

  <ol>
    <li>Télécharge et installe Git depuis <a href="https://git-scm.com/downloads" target="_blank" rel="noopener">git-scm.com/downloads</a>.</li>
    <li>Après l’installation, ouvre <b>Git Bash</b> (Windows) ou ton terminal (macOS/Linux).</li>
    <li>Configure ton identité (c’est comme signer ton nom sur ton journal) :</li>
    <div class="cmd-row">
      <pre class="cmd language-bash"><code>git config --global user.name "Ton Nom"
git config --global user.email "ton.email@example.com"</code></pre>
    </div>
  </ol>

  <h3>3. Installer Visual Studio Code (VS Code) — ton atelier</h3>
  <p>
    VS Code est un éditeur de code très utilisé : il affiche ton code proprement, propose l’autocomplétion, permet d’exécuter des commandes et d’installer des extensions (comme des petites applications pour ton atelier).
    <i>Analogie :</i> si Python est la machine à café, VS Code est la cuisine équipée avec un plan de travail et des tiroirs pleins d’outils.
  </p>

  

    <li>
      <b>Exécuter ton script depuis VS Code</b> :
      <ol>
        <li>Ouvre <code>main.py</code> dans VS Code.</li>
        <li>Soit tu utilises le terminal intégré et tapes <code>python main.py</code>, soit tu utilises la petite flèche « Run » en haut à droite (si l’extension Python est active).</li>
      </ol>
    </li>

    <li>
      <b>Version Web (option rapide)</b> :
      <p>Si tu veux écrire du code sans installer VS Code, tu peux essayer <a href="https://vscode.dev" target="_blank" rel="noopener">VS Code pour le Web</a>. C’est pratique pour éditer rapidement, mais il manque certaines fonctionnalités (terminal local complet, exécution locale automatique). </p>
    </li>
  </ol>
  <p>
