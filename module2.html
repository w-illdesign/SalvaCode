Introduction aux variables

Salut ! On commence très simplement. Si un mot technique apparaît, il est défini immédiatement. Pas d’exercices pour l’instant — juste des explications calmes et des petits exemples faciles à suivre.

1. Qu’est-ce qu’une variable ?

Imagine une boîte dans laquelle tu peux mettre quelque chose. Sur la boîte, tu colles une étiquette pour te souvenir de ce qu’il y a dedans.

  • La boîte représente la variable.
  • L’étiquette est le nom de la variable.
  • Le contenu est la valeur que contient la variable.
Définition — Variable
Un nom qui désigne un emplacement (dans la mémoire de l’ordinateur) où on peut stocker une information pour la réutiliser ou la changer.

Exemple simple (vie courante) : tu as une boîte étiquetée « clés ». Quand tu veux tes clés, tu vas directement à la boîte « clés ». Une variable, c’est pareil : au lieu de retenir une valeur compliquée, tu retiens un nom et tu regardes ce qu’il y a dedans.

2. Comment on crée une variable (en Python) — pas à pas

En Python, pour créer une variable on écrit : nom = valeur. Le signe = s’appelle affectation. Ici, il veut dire « mets cette valeur dans cette boîte ».

Exemples :

nom = "Alice"

Que fait cette ligne ?

  1. À gauche du = : le nom nom (c’est l’étiquette).
  2. À droite : la valeur "Alice" (le texte).
  3. Python met la valeur "Alice" dans la variable nom.

Autre exemple :

age = 25
Définition — Affectation
Action de donner une valeur à une variable (utiliser =).
Définition — Identifiant
Le nom que tu donnes à la variable (par ex. nom, age).

3. Règles simples pour nommer une variable

Quand tu choisis un nom, suis ces règles simples :

  • Commence par une lettre ou par un underscore _. Ne commence pas par un chiffre.
  • Pas d’espaces : utilise _ pour séparer les mots (ex. nom_utilisateur).
  • Les majuscules et minuscules sont différentes : Nom et nom sont deux choses distinctes.
Définition — Sensible à la casse (case-sensitive)
Cela veut dire que A et a sont différents.

Choisis des noms qui ont du sens : prix_total est plus clair que x. Ces règles rendent ton code plus lisible et évitent des erreurs.

4. La variable peut changer (réaffectation) — expliquée lentement

Une fois qu’une variable contient quelque chose, tu peux lui donner une autre valeur. On appelle cela réaffecter la variable.

Exemple :

compteur = 1      # la variable compteur contient 1
compteur = compteur + 1  # on prend la valeur actuelle, on ajoute 1, on remet le résultat dans compteur

Après la seconde ligne, compteur contient 2.

Pense-y comme : tu as une boîte étiquetée « compteur » contenant une balle numérotée 1. Tu prends la balle, tu la remplaces par une autre balle numérotée 2, mais tu laisses l’étiquette.

Autre exemple :

message = "Bonjour"
message = "Bonsoir"   # on remplace le texte

5. Exemple concret expliqué ligne par ligne

Supposons que tu veux afficher un message de salutation qui utilise le prénom d’une personne :

prenom = "Samir"
print("Bonjour " + prenom + "!")

Explication :

  1. prenom = "Samir" — on crée la variable prenom contenant le texte "Samir".
  2. print("Bonjour " + prenom + "!") — print() affiche du texte ; on concatène (colle) trois morceaux : "Bonjour ", la valeur de prenom, et "!". Résultat : Bonjour Samir!
Définition — Concaténation
Action de coller des morceaux de texte ensemble.

6. Un modèle mental simple (mémoire)

Imagine la mémoire de l’ordinateur comme une rangée de casiers. Chaque casier a une étiquette (nom de la variable). La valeur est l’objet mis dans le casier. Quand tu veux lire ou modifier l’information, tu vas au casier correspondant.

Utilité : ce modèle t’aide à comprendre pourquoi on donne des noms aux variables : pour retrouver facilement les informations.

7. Erreurs fréquentes expliquées calmement

Erreurs courantes et comment les corriger :

Oublier les guillemets pour un texte

nom = Alice    # Python va chercher une variable appelée Alice — ça provoque une erreur
nom = "Alice"  # Correct : "Alice" est un texte

En Python, les textes doivent être entourés de guillemets "..." ou '...'.

Mélanger texte et nombre sans conversion

"3" + 2        # Erreur : on ne peut pas additionner un texte et un nombre
int("3") + 2   # Correct : on transforme le texte "3" en nombre 3 puis on ajoute 2
Définition — Conversion (ou cast)
Action de transformer une valeur d’un type à un autre (par ex. de texte en nombre).

Utiliser un mot réservé comme nom

for = 5   # Erreur : "for" est un mot réservé

Astuce pratique : si Python affiche une erreur, commence par lire la première ligne du message d’erreur — elle indique souvent quelle ligne ou quelle variable pose problème.

8. Récapitulatif très clair (ce qu’il faut garder en tête)

  • Une variable = une boîte + une étiquette + un contenu.
  • Tu crées une variable en écrivant nom = valeur. Le signe = signifie affectation.
  • Tu peux changer la valeur plus tard (réaffectation).
  • Donne des noms clairs et respecte les règles de nommage (pas d’espace, pas de chiffre au début, sensible à la casse).
  • Fais attention aux guillemets pour les textes et aux conversions quand tu mélanges types différents.

Types de données

Ici on détaille ce qu’on peut mettre dans les boîtes (les variables). Pour chaque type : définition simple, analogie du quotidien, exemple en Python, erreurs fréquentes et conseils.

1. Qu’est-ce qu’un type de données ?

Définition : le type d’une valeur décrit sa nature (nombre, texte, vrai/faux, collection...).

Analogie : une boîte à outils : chaque outil est fait pour une tâche précise. Les types te disent quelles opérations sont adaptées — on n’utilise pas un tournevis pour scier.

Pourquoi c’est important : le type détermine ce qu’on peut faire avec une valeur. Mélanger des types sans conversion correcte provoque souvent des erreurs.

Mot-clé — Type
Catégorie décrivant la nature d’une donnée (ex. entier, texte, liste).

2. Entiers (int)

Définition
Nombres entiers, sans partie décimale (ex. 0, 5, -10).
Analogie
Compter des pommes : tu as 2 pommes, pas 2,5 (les entiers comptent des objets discrets).
Usage courant
Âge, compteurs, indices dans une liste, nombre d’éléments.
age = 30
print(type(age))   # <class 'int'>
print(age + 5)     # 35
print(7 // 2)      # 3  (division entière)
print(7 / 2)       # 3.5 (division normale -> float)

Astuce : pour compter des groupes entiers, utilise int et la division entière //.

3. Nombres décimaux (float)

Définition
Nombres avec partie décimale (ex. 3.14, -0.5).
Analogie
Mesures (litres, kilogrammes, prix) : on a besoin de précision, souvent fractional.
Usage courant
Prix, mesures physiques, moyennes, calculs scientifiques.
prix = 1.75
print(type(prix))  # <class 'float'>
print(prix * 2)    # 3.5
print(0.1 + 0.2)   # -> 0.30000000000000004 (imprécision binaire courante)

Conseils : - Les float peuvent présenter de petites imprécisions (stockage binaire). - Pour comparer des floats, évite == ; utilise une tolérance : abs(a - b) < 1e-9. - Pour l’argent, préfère decimal.Decimal si tu as besoin d’exactitude.

4. Texte (str)

Définition
Suite de caractères entourée de guillemets (ex. "Bonjour", 'Alice').
Analogie
Une étiquette ou une note écrite : on lit l’information, on ne la calcule pas.
Opérations courantes
Concaténation, longueur (len()), recherche, transformation (.upper()).
nom = "Alice"
print(type(nom))          # <class 'str'>
print("Bonjour " + nom)   # Bonjour Alice
print(f"Bonjour {nom}")   # f-string (plus lisible)
print(len(nom))           # 5

Erreur fréquente : "123" est du texte, pas le nombre 123. Pour calculer, convertis : int("123").

5. Booléens (bool)

Définition
Valeurs logiques : True ou False.
Analogie
Un interrupteur : allumé (= True) ou éteint (= False).
Usage courant
Conditions (if), contrôles d’accès, résultats de comparaisons.
est_adulte = True
a_permis = False
if est_adulte and a_permis:
    print("Peut conduire")

Note pratique : en Python, de nombreux objets ont une valeur de vérité : bool(0) → False, bool("") → False, bool([1]) → True.

6. Valeur spéciale None

Définition
Représente l’absence de valeur (type NoneType): rien pour l’instant.
Analogie
Une boîte vide avec un post-it "vide" : la boîte existe mais elle ne contient rien d’utile.
Usage courant
Initialiser une variable avant d’avoir un résultat, marquer l’absence d’information.
resultat = None
if resultat is None:
    print("Pas encore calculé")

Astuce : pour tester None, préfère is None plutôt que == None.

7. Types composés — aperçu pratique

7.1 list (liste)

Définition : collection ordonnée et modifiable : [a, b, c].

Analogie : une étagère de cuisine avec des cases : l’ordre compte et tu peux ajouter/enlever des objets.

courses = ["pain", "lait", "oeufs"]
courses.append("bananes")
print(courses[0])  # "pain"

Astuce : utile pour stocker une suite d’éléments à parcourir.

7.2 tuple (tuple)

Définition : collection ordonnée et immutable : (x, y).

Analogie : une photo de l’étagère : tu vois l’état à un instant T, mais tu ne peux pas bouger les objets sur la photo.

coord = (48.8566, 2.3522)  # coordonnées fixes
print(coord[0])

7.3 dict (dictionnaire)

Définition : ensemble de paires clé → valeur, non ordonné (ex. {"nom":"Ali", "age":20}).

Analogie : un carnet d’adresses : tu donnes une clé (nom) pour obtenir la valeur (numéro).

personne = {"nom": "Awa", "age": 20}
print(personne["nom"])        # "Awa"
print(personne.get("prenom")) # None si la clé n'existe pas

Astuce : utilise .get() si la clé peut manquer.

7.4 set (ensemble)

Définition : collection non ordonnée d’éléments uniques.

Analogie : une boîte de timbres : chaque timbre (élément) n’apparaît qu’une seule fois.

A = {1, 2, 3}
B = {3, 4, 5}
print(A | B)  # union {1,2,3,4,5}
print(A & B)  # intersection {3}

8. Mutable vs Immutable (implications pratiques)

Mutable : on peut modifier l’objet après sa création (list, dict).

Immutable : l’objet ne change pas ; on crée une nouvelle valeur pour "modifier" (int, str, tuple).

l = [1,2,3]
l.append(4)     # modifie la même liste

s = "bonjour"
s2 = s.upper()  # crée une NOUVELLE chaîne ; s reste inchangé

Pourquoi c’est utile : les immutables sont sûrs à partager (pas d’effets de bord). Les mutables sont pratiques mais demandent de la prudence quand plusieurs fonctions les manipulent.

9. Opérations courantes & bonnes pratiques

  • Nombres : + - * / // % **
  • Strings : concaténation (+), répétition (*), slicing (s[1:3]), méthodes (.upper())
  • Booléens : and, or, not
  • Listes / dicts : indexation, .append(), .pop(), test d'appartenance (in)

Conseils pratiques : - Préfère les f-strings pour formater : f"Prix : {prix} €". - Fais toujours les conversions explicitement : int(), str(), float(). - Pour les montants financiers, évite les floats ; utilise decimal.Decimal quand l’exactitude est critique.

10. Conversion (casting)

Définition
Transformer une valeur d’un type à un autre : int(), float(), str(), bool().
Analogie
Changer de contenant ou d’unité (verser du riz d’un sac dans un bocal, convertir 100 centimes → 1 euro).
int("42")      # 42
float("3.14")   # 3.14
str(10)         # "10"

Piège : int("3.14") lèvera ValueError. Solution : int(float("3.14")) ou valider / utiliser try/except.

11. Vérifier le type : type() vs isinstance()

x = 3.14
print(type(x))                # <class 'float'>
print(isinstance(x, float))   # True

Remarque : isinstance est souvent préférable pour vérifier un type dans du code, car il gère l’héritage et est plus souple.

12. Erreurs courantes & comment les éviter

  • Concaténer str et int : "Total : " + 3 → TypeError. Corriger : "Total : " + str(3) ou f"Total : {3}".
  • Conversion invalide : int("bonjour") → ValueError. Utiliser try/except ou valider l’entrée.
  • Comparer des floats : éviter ==, utiliser une tolérance (abs(a-b) < epsilon).
  • Effets de bord : attention aux objets mutables partagés entre fonctions ; documente ou copie si nécessaire (lst.copy()).

13. Modèles mentaux (mnémotechniques)

  • Boîte + étiquette → variable + nom (on met un objet dans un emplacement identifié).
  • Nature / Type → chaussure vs bouteille (les opérations diffèrent).
  • Mutable / Immutable → tableau blanc vs feuille imprimée.
  • Conversion → changer d’unité (ml ↔ L) — parfois impossible.

14. Résumé

En Python, chaque valeur a un type qui détermine ce qu’on peut en faire. Maîtriser les types de base (int, float, str, bool), les collections (list, tuple, dict, set) et la valeur spéciale None t’aide à écrire du code robuste et à déboguer rapidement.

En résumé, chaque type de données en Python a ses caractéristiques et son comportement particulier. Certains (comme int, float, str ou tuple) sont immuables, c’est-à-dire qu’ils ne changent pas une fois créés. D’autres (comme list, dict ou set) sont mutables et peuvent être modifiés au cours du programme.

On peut comparer cela à des objets de la vie courante : une tuple ressemble à un ticket de caisse imprimé (il ne change plus après impression), tandis qu’une list ressemble à un cartable dans lequel on peut ajouter ou retirer des cahiers à volonté.

Bien comprendre les types de données est la clé pour écrire des programmes fiables. C’est un peu comme connaître les ingrédients de base avant de commencer à cuisiner : si tu sais exactement ce que tu manipules, tu pourras créer des recettes (programmes) beaucoup plus facilement.
