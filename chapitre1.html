Module 1.Installation des outils

Pour ceux qui ont un ordinateur

1. Installer Python

  1. Va sur python.org/downloads
  2. Clique sur Download Python 3.x.
  3. Lance le fichier d’installation.
  4. Important : Coche impérativement Add Python to PATH avant de cliquer sur Install Now. ⚠️ Sans cela, Git Bash ne trouvera pas la commande python.
  5. Termine l’installation.
  6. Sur macOS/Linux : utilise Homebrew (macOS) ou ton gestionnaire de paquets (apt, yum) pour installer Python.

2. Installer Git (Git Bash)

  1. Télécharge Git : git-scm.com/downloads
  2. Installe Git en gardant les options par défaut.
  3. Tu auras maintenant Git Bash dans ton menu Démarrer.

3. Vérifier l’installation

Ouvre Git Bash et tape :

# On vérifie que Python et Git sont bien installés.
python --version
git --version

Tu dois voir quelque chose comme Python 3.x.x et git version x.y.z.

Si python n’est pas reconnu, retourne à l’étape 1 et assure-toi d’avoir coché Add to PATH.

4. Créer ton premier dossier projet

# On se déplace vers le dossier "Documents".
cd /c/Users/TonNom/Documents
# On crée un nouveau dossier pour notre projet et on y entre.
mkdir mon_projet && cd mon_projet
# On vérifie notre position.
pwd

5. Écrire et lancer ton premier script

Ouvre VS Code, clique sur Ouvrir un dossier, et choisis mon_projet.

Crée un fichier main.py et écris :

# La commande print() affiche un message.
print("Bonjour depuis Git Bash !")

Sauvegarde (Ctrl+S).

Dans Git Bash, lance :

# On lance le script Python qu'on vient de créer.
python main.py

Résultat attendu : Bonjour depuis Git Bash !

Pour ceux qui ont un smartphone (Android)

Pour coder depuis un smartphone, nous utiliserons Termux, un terminal qui émule un environnement Linux.

1. Installer Termux

Télécharge l'application Termux depuis F-Droid, car la version du Play Store n'est plus mise à jour. Installe-la, tu auras une invite de commande prête à l'emploi.

2. Mettre à jour les paquets

# On met à jour Termux et ses outils.
pkg update && pkg upgrade -y

3. Installer Python

# On installe le langage Python.
pkg install python

4. Vérifier l’installation

# On vérifie que Python est bien installé.
python --version

Tu devrais voir la version de Python que tu viens d'installer.

5. Écrire et lancer ton premier script

Tu peux utiliser un éditeur de texte simple comme nano qui est inclus.

# On crée un nouveau dossier pour notre projet et on y entre.
mkdir mon_projet
cd mon_projet
# On crée et on ouvre le fichier pour écrire notre code.
nano main.py

Écris ton code :

# La commande print() affiche un message.
print("Bonjour depuis Termux !")

Sauvegarde et quitte en appuyant sur Ctrl + X, puis Y pour confirmer, et Entrée.

Lance ton script :

# On lance notre script Python.
python main.py

Le résultat attendu est : Bonjour depuis Termux !

3. Problèmes courants

bash: commande introuvable

Cela signifie que le terminal ne trouve pas la commande que tu as tapée.

  • Vérifie l'orthographe : Les fautes de frappe sont la cause la plus fréquente.
  • Vérifie la casse : Les noms de commandes sont sensibles à la casse.
  • Vérifie ta position : Utilise la commande pwd pour savoir où tu es. Utilise ls pour lister le contenu du dossier actuel.

FileNotFoundError: [Errno 2] No such file or directory

Le programme ne trouve pas le fichier ou le dossier spécifié.

  • Vérifie l'orthographe : Les fautes de frappe sont la cause la plus fréquente. Vérifie deux fois le nom du dossier ou du fichier.
  • Vérifie la casse : Les noms de fichiers et de dossiers sont sensibles à la casse (mon_projet n'est pas la même chose que Mon_Projet).
  • Vérifie ta position : Utilise la commande pwd pour savoir où tu es. Utilise ls pour lister le contenu du dossier actuel et t'assurer que le fichier ou le dossier que tu cherches est bien là.

Problèmes de syntaxe et de code

SyntaxError: invalid syntax

Le code que tu as écrit ne respecte pas les règles du langage Python.

  • Vérifie que tes guillemets sont bien présents et par paires (" ou ').
  • Vérifie que tes parenthèses sont bien fermées (()).
  • La plupart des éditeurs de code (comme VS Code) te montreront la ligne où se trouve l'erreur.

IndentationError: expected an indented block

En Python, l'indentation (les espaces en début de ligne) est très importante.

  • Fais très attention à l'alignement de ton code.
  • Utilise toujours des espaces plutôt que des tabulations.
