Module 1.Installation des outils

Pour ceux qui ont un ordinateur

Avant de commencer — quelques mots simples :

Un terminal (ou « invite de commandes ») est un endroit où tu tapes des commandes (des instructions) pour dire à l'ordinateur ce qu'il doit faire. Un fichier est comme une feuille de papier (par ex. main.py), et un dossier est comme une chemise où on range des fichiers.

1. Installer Python — qu’est-ce que c’est ?

Python est un langage de programmation : c’est l’outil qui va lire et exécuter les instructions que tu écris (tes programmes). On commence par l’installer sur ton ordinateur.

  1. Télécharger Python :
    Va sur python.org/downloads. Tu verras un gros bouton Download Python 3.x — clique dessus.
  2. Exécuter l’installeur :
    • Sur Windows : double-clic sur le fichier que tu as téléchargé (.exe).
    • Sur macOS : ouvre le paquet .pkg et suis les instructions.
    • Sur Linux : tu peux souvent installer depuis le gestionnaire de paquets (voir dessous).
  3. Important — Windows : avant d’installer, coche la case Add Python to PATH dans la fenêtre d’installation.

    Pourquoi ? PATH est une liste d'endroits où l'ordinateur cherche les programmes. Si Python n’est pas dans PATH, tu ne pourras pas lancer la commande python depuis le terminal.

  4. Terminer l’installation et fermer l’installeur. Si on te demande de redémarrer le terminal, accepte (ou ferme/rouvre le terminal).
  5. Linux (Debian/Ubuntu) — si tu préfères installer avec les commandes : 
    sudo apt update
sudo apt install -y python3 python3-pip

    Sur macOS, tu peux utiliser Homebrew (optionnel) : brew install python.

Comment vérifier que Python est bien installé ? 
python --version        # souvent fonctionne sur Windows si Add to PATH a été coché
python3 --version       # souvent utilisé sur macOS / Linux
py -3 --version         # alternative sur certaines installations Windows

Tu dois voir quelque chose comme : Python 3.11.4 (le numéro peut être différent).

2. Installer Git (et Git Bash)

Git est un outil pour sauvegarder l’historique de tes fichiers (on l’utilise beaucoup en programmation). Git Bash est un terminal convivial pour Windows fourni avec Git.

  1. Télécharge Git depuis git-scm.com/downloads et installe-le (tu peux garder les options par défaut).
  2. Après installation, ouvre Git Bash (sur Windows) ou ton terminal habituel (macOS/Linux).
  3. Configure ton nom et ton e-mail — ces informations servent si tu veux sauvegarder des versions sur des services comme GitHub :
    4. git config --global user.name "Ton Nom"
git config --global user.email "ton.email@example.com"

    Ces commandes disent à Git « qui travaille sur ces fichiers ». Elles n’envoient rien sur Internet, c’est juste pour ton ordinateur.

Petite explication : Git vs GitHub

Git est le logiciel sur ton ordinateur. GitHub (ou GitLab) est comme un service en ligne où tu peux stocker une copie de tes projets pour y accéder depuis d'autres ordinateurs. Pour commencer, tu n’as pas besoin de GitHub — tu peux apprendre Git localement.

3. Vérifier que tout fonctionne (Python, pip, Git)

Maintenant, allons dans le terminal (Git Bash) et vérifions les commandes. Tape exactement : 

python --version
python -m pip --version
git --version

Résultat attendu (exemples) :

  • Python 3.x.x
  • pip x.x.x from ...
  • git version x.y.z

Si une commande n’est pas reconnue :

  1. Relis l’étape d’installation et vérifie que tu as bien coché “Add to PATH” (Windows).
  2. Sur Windows, essaie la commande py -3 --version — parfois Python répond via le lanceur py.
  3. Si besoin, relance l’ordinateur ou ferme/réouvre le terminal.

4. Créer ton premier dossier projet — pas à pas

On va créer un dossier (un emplacement) pour ranger le projet. C’est très simple :

# Aller dans le dossier "Documents" (toutes plateformes) :
cd "$HOME/Documents"

# Créer un dossier nommé "mon_projet" et entrer dedans :
mkdir mon_projet && cd mon_projet

# Afficher où l'on est :
pwd

# Lister les fichiers (devrait être vide pour un début) :
ls

Explication ligne par ligne :

  1. cd "$HOME/Documents" : se déplace dans le dossier Documents de ton compte (HOME signifie ton dossier personnel).
  2. mkdir mon_projet : crée un dossier appelé mon_projet.
  3. cd mon_projet : entre dans le dossier que tu viens de créer.
  4. pwd : montre le chemin complet du dossier où tu es (utile pour être sûr).
  5. ls : affiche les fichiers et dossiers présents.
Astuce Windows (si Git Bash ne pointe pas vers Documents)

Tu peux utiliser :

cd /c/Users/TonNom/Documents
# Remplace "TonNom" par ton nom d'utilisateur Windows (celui que tu vois dans l'explorateur de fichiers)

5. Installer Vs code

  1. Télécharger VS Code depuis le site officiel : code.visualstudio.com/download. :contentReference[oaicite:0]{index=0}
    2. 2. Choisir la bonne version :
    • Windows : User Installer (pour un utilisateur) ou System Installer (si tu veux l’installer pour tous les comptes).
    • macOS : ouvre le .zip / .dmg et glisse VS Code dans Applications.
    • Linux : utilise le package .deb (Debian/Ubuntu) ou .rpm (Fedora) ou installe via le gestionnaire de paquets. Exemple : télécharger le .deb puis sudo apt install ./vscode.deb. :contentReference[oaicite:1]{index=1}
  2. Exemples pas à pas (Windows) :
    1. Double-clic sur le fichier téléchargé (User Installer).
    2. Accepte la licence et garde les options par défaut (tu peux cocher « Add to PATH » si proposé).
    3. Termine l’installation et lance VS Code depuis le menu Démarrer.
  3. Ouvrir ton dossier de projet dans VS Code :

    Dans VS Code : Fichier → Ouvrir un dossier puis choisis mon_projet. Cela permet à VS Code de voir tous les fichiers du projet (comme ouvrir la chemise pour voir toutes les feuilles).

  4. Ouvrir un terminal intégré :

    Dans VS Code : Terminal → Nouveau terminal. Le terminal s’ouvre en bas de VS Code et fonctionne comme Git Bash / ton terminal habituel. Tu peux taper les mêmes commandes ici (ex. python main.py).

  5. Installer l’extension Python (conseillé) :

    Va dans l’onglet Extensions (icône en forme de blocs ou Ctrl+Shift+X), cherche « Python » (par Microsoft) et clique sur Installer. Cette extension aide pour l’autocomplétion, le diagnostic des erreurs et le débogage.

    Analogie : l’extension Python, c’est comme une loupe et un guide posé sur ton plan de travail : elle t’aide à repérer les erreurs et à travailler plus vite.

5. Écrire et lancer ton premier script Python — tout expliqué

On va créer un petit fichier Python qui écrit un message dans le terminal. C’est l’étape la plus excitante !

  1. Ouvrir Visual Studio Code (VS Code) :
    • Si VS Code est installé : ouvre-le, puis Fichier → Ouvrir un dossier et choisis le dossier mon_projet que tu as créé.
    • Si tu n’as pas VS Code, tu peux utiliser un éditeur simple (Notepad sous Windows, TextEdit sur macOS en mode texte, ou l’éditeur nano depuis le terminal).
  2. Créer le fichier :

    Dans VS Code, clique sur l’icône « nouveau fichier » et nomme-le main.py (l’extension .py indique que c’est un fichier Python).

  3. Écris ce code simple dans main.py :
    4. # Ceci affiche un message dans le terminal.
print("Bonjour depuis Git Bash !")
  4. Enregistre (sauvegarde) le fichier : Ctrl+S (Windows) ou Cmd+S (macOS).
  5. Exécuter le script (dans Git Bash ou terminal ouvert dans le dossier mon_projet) :
    6. python main.py      # ou: python3 main.py / py -3 main.py
  6. Résultat attendu : tu vois dans le terminal la ligne suivante :
    7. Bonjour depuis Git Bash !

Si rien n’apparaît ou si tu as une erreur :

  1. Vérifie que tu es dans le bon dossier : tape pwd puis ls et assure-toi que main.py est listé.
  2. Si l'ordinateur répond python: command not found, reviens à l’étape d’installation de Python et vérifie la case Add to PATH (Windows) ou utilise python3 sur macOS/Linux.
  3. Si tu as une erreur de syntaxe, regarde bien les guillemets et les parenthèses — VS Code montre souvent la ligne où il y a un problème.
Exemples simples à essayer ensuite 
print(2 + 2)
name = input("Comment tu t'appelles ? ")
print("Bonjour", name)

Ces petits essais t’aident à comprendre comment Python lit et exécute les lignes une par une.

Derniers conseils pour un débutant :

  1. Ne t’inquiète pas des erreurs — elles arrivent toujours. Lis ce que l’ordinateur affiche : il donne souvent un indice précis (ligne, type d’erreur).
  2. Prends ton temps : fais une étape à la fois (installer → vérifier → créer dossier → écrire fichier → exécuter).
  3. Si quelque chose bloque, nous serons là pour t'aider et t'assister.

Pour ceux qui ont un smartphone (Android )

Peut-être que tu n’as pas encore d’ordinateur, mais seulement un téléphone Android. Pas de problème ! Tu peux quand même apprendre à coder directement depuis ton smartphone.

Pour cela, nous allons utiliser une application qui s’appelle Termux. Mais alors, c’est quoi Termux ? 🤔

Imagine que ton téléphone est comme une maison remplie de pièces fermées à clé. Tu as plein de choses à l’intérieur (photos, vidéos, applications…), mais tu ne vois pas toutes les portes. Termux, c’est comme une clé magique qui ouvre une porte secrète, donnant accès à une salle spéciale appelée terminal.

Qu’est-ce qu’un terminal ?

Un terminal est un écran où tu écris des commandes en texte pour donner des ordres à ton appareil. C’est comme discuter avec ton téléphone en langage direct, sans passer par des boutons ou des icônes. Exemple : au lieu d’appuyer sur un bouton pour installer une application, tu écris une commande et ton téléphone le fait.

Dans ce terminal, tu ne cliques pas avec les doigts comme sur WhatsApp ou YouTube. Ici, tu écris des ordres avec du texte, et ton téléphone obéit. C’est comme si ton smartphone devenait ton employé et que tu pouvais lui dire : « Hé, range-moi ce dossier », « écris-moi ça », « calcule ça »…

À quoi sert Termux ?
  • À transformer ton téléphone en un ordinateur miniature.
  • À apprendre à coder même sans PC.
  • À exécuter des programmes et tester directement tes idées.

1. Installer Termux

D’abord, il faut télécharger Termux. ⚠️ Attention : ne le prends pas sur le Play Store, car cette version n’est plus à jour (comme un cahier rempli de vieilles leçons). Il faut aller sur F-Droid, qui est comme une librairie en ligne pour télécharger des applications gratuites et fiables.

Qu’est-ce que F-Droid ?

F-Droid est une boutique d’applications gratuite et ouverte (un peu comme un marché bio 🥦 pour les applis). Contrairement au Play Store, tout y est libre et transparent : tu peux y trouver des applications sûres, sans pub et mises à jour régulièrement.

Une fois Termux installé et ouvert, tu verras un écran noir avec du texte blanc. Pas de panique 😅 ! Cet écran, c’est comme si tu parlais à ton téléphone en langage de commandes.

2. Mettre à jour les paquets

Comme pour ton téléphone, où tu fais des mises à jour pour avoir les dernières fonctionnalités et corriger les bugs, ici aussi il faut mettre à jour Termux. C’est un peu comme remplir ton frigo avant de commencer à cuisiner : il faut être sûr que tout est frais. 

# On met à jour Termux et ses outils.
pkg update && pkg upgrade -y
>Qu’est-ce qu’un paquet ?

Un paquet est comme une petite boîte qui contient un logiciel ou un outil. Quand on dit « mettre à jour les paquets », ça veut dire : installer les nouvelles versions de tous les outils pour qu’ils soient prêts et sécurisés.

Tape cette commande dans Termux et appuie sur "Entrée". Attends que ça finisse, ça peut prendre quelques minutes selon ta connexion.

3. Installer Python

Maintenant, il est temps d’installer Python, notre langage de programmation. Imagine que Python est comme une langue étrangère : en l’installant, ton téléphone apprend à parler cette langue. Ensuite, toi tu vas apprendre à donner des phrases à ton téléphone pour qu’il obéisse (programmes). 

# On installe le langage Python.
pkg install python

Tape cette commande et valide. Ton téléphone va télécharger et installer Python. C’est comme si tu ajoutais un dictionnaire dans ton sac pour apprendre une nouvelle langue.

Une fois que c’est terminé, tu es prêt à écrire tes premiers programmes !

Tu verras que l’écran de Termux est noir avec du texte blanc (parfois vert). Cela peut sembler étrange au début, mais c’est la porte d’entrée du monde des programmeurs. Beaucoup de grands développeurs travaillent encore aujourd’hui avec ce genre d’outils !

4. Vérifier l’installation

# On vérifie que Python est bien installé.
python --version

Après avoir installé Python, il faut toujours vérifier si l’installation a bien fonctionné. Pour ça, on tape la commande python --version dans Termux.

Que veut dire --version ?

L’option --version demande à un programme d’afficher sa version actuelle. C’est comme demander à quelqu’un : « Quel âge as-tu ? ». Ici, on demande à Python : « Quelle est ta version ? ».

Si tout s’est bien passé, tu verras apparaître un texte comme par exemple : 

Python 3.11.7

Ici, "3.11.7" est la version de Python. Toi, tu peux avoir une version un peu différente (par exemple 3.12.2), et ce n’est pas grave : l’important est que quelque chose s’affiche !

Imagine que tu viens d’acheter un nouveau téléphone. Avant de l’utiliser, tu l’allumes pour vérifier qu’il fonctionne et qu’il a bien une batterie. Eh bien ici, c’est pareil : on installe Python, puis on le "réveille" avec python --version pour s’assurer qu’il est bien là et prêt à travailler.

5. Écrire et lancer ton premier script

Maintenant que Python est installé et reconnu, on va écrire notre tout premier programme. C’est un peu comme envoyer ton premier SMS 📱 avec un nouveau téléphone : on vérifie qu’il fonctionne vraiment.

Pour écrire du code, on utilise un éditeur de texte. Termux inclut déjà nano, qui est simple et pratique. 

# On crée un nouveau dossier pour notre projet et on y entre.
mkdir mon_projet
cd mon_projet

# On crée et on ouvre le fichier pour écrire notre code.
nano main.py
Qu’est-ce qu’un script ?

Un script est simplement un fichier texte qui contient du code que Python va lire et exécuter. Dans notre cas, main.py est notre premier script : un petit programme que nous allons lancer.

Dans l’éditeur nano, tape ce code : 

# La commande print() sert à afficher du texte à l’écran.
print("Bonjour depuis Termux !")
Qu’est-ce que print() ?

La fonction print() est l’une des plus utilisées en Python. Elle sert à afficher du texte ou des résultats à l’écran. Exemple : print(2 + 3) affichera 5.

Ensuite, il faut enregistrer ton fichier :

  • Ctrl + X (pour quitter)
  • Y (pour confirmer l’enregistrement)
  • Entrée (pour valider le nom du fichier)

Qu’est-ce que nano ?

nano est un éditeur de texte inclus dans Termux. Il s’ouvre directement dans ton terminal, et te permet de taper du code ou des notes. Il est léger, simple et parfait pour débuter.

Maintenant, lançons ton programme : 

# On exécute notre script Python.
python main.py

Résultat attendu à l’écran : 

Bonjour depuis Termux !
Qu’est-ce que Python exactement ?

Python est un langage de programmation populaire, simple et lisible. Il est utilisé dans de nombreux domaines : intelligence artificielle, web, jeux vidéo, analyse de données, etc. Sa simplicité le rend idéal pour apprendre à programmer.

Bravo ! Tu viens d’exécuter ton tout premier programme Python sur Android grâce à Termux. À partir de là, tu pourras écrire des scripts de plus en plus complexes.

Commandes de navigation et gestion de fichiers

1️ Navigation dans les dossiers

# Liste les fichiers et dossiers dans le dossier actuel
ls
# Liste détaillée avec taille, date et permissions
ls -l
# Entrer dans un dossier
cd nom_du_dossier
# Revenir au dossier précédent
cd ..
# Afficher le chemin complet du dossier actuel
pwd
# Afficher l’arborescence des dossiers et fichiers
tree
# Copier un projet Git depuis Internet
git clone url_du_repo
# Se déplacer sur une autre branche Git
git checkout nom_de_branche
# Voir toutes les branches disponibles
git branch

2️ Manipulation des fichiers

# Crée un nouveau fichier vide
touch nom_du_fichier
# Ouvre un fichier pour l’éditer
nano nom_du_fichier
# Affiche le contenu d’un fichier
cat nom_du_fichier
# Supprime un fichier (attention ⚠️)
rm nom_du_fichier
# Renommer ou déplacer un fichier
mv fichier1 fichier2
# Copier un fichier
cp fichier1 fichier2
# Crée un nouveau dossier
mkdir nom_du_dossier
# Supprime un dossier vide
rmdir nom_du_dossier
# Supprime un dossier et tout son contenu (attention ⚠️)
rm -r nom_du_dossier
# Affiche les premières lignes d’un fichier
head nom_du_fichier
# Affiche les dernières lignes d’un fichier
tail nom_du_fichier
# Copier un dossier et tout son contenu
cp -r dossier1 dossier2

Avec ces commandes de base, tu peux déjà naviguer, créer et lancer tes premiers scripts en toute autonomie. Le plus important maintenant est de pratiquer un peu chaque jour : même des petits programmes comme afficher ton prénom, faire une addition ou afficher une liste. Petit à petit, tu deviendras à l’aise et tu pourras créer des projets plus grands !

3. Problèmes courants

bash: commande introuvable

Cela signifie que le terminal ne trouve pas la commande que tu as tapée.

  • Vérifie l'orthographe : Les fautes de frappe sont la cause la plus fréquente.
  • Vérifie la casse : Les noms de commandes sont sensibles à la casse.
  • Vérifie ta position : Utilise la commande pwd pour savoir où tu es. Utilise ls pour lister le contenu du dossier actuel.

FileNotFoundError: [Errno 2] No such file or directory

Le programme ne trouve pas le fichier ou le dossier spécifié.

  • Vérifie l'orthographe : Les fautes de frappe sont la cause la plus fréquente. Vérifie deux fois le nom du dossier ou du fichier.
  • Vérifie la casse : Les noms de fichiers et de dossiers sont sensibles à la casse (mon_projet n'est pas la même chose que Mon_Projet).
  • Vérifie ta position : Utilise la commande pwd pour savoir où tu es. Utilise ls pour lister le contenu du dossier actuel et t'assurer que le fichier ou le dossier que tu cherches est bien là.

Problèmes de syntaxe et de code

SyntaxError: invalid syntax

Le code que tu as écrit ne respecte pas les règles du langage Python.

  • Vérifie que tes guillemets sont bien présents et par paires (" ou ').
  • Vérifie que tes parenthèses sont bien fermées (()).
  • La plupart des éditeurs de code (comme VS Code) te montreront la ligne où se trouve l'erreur.

IndentationError: expected an indented block

En Python, l'indentation (les espaces en début de ligne) est très importante.

  • Fais très attention à l'alignement de ton code.
  • Utilise toujours des espaces plutôt que des tabulations.
